01 / Features

Two engines. One uncompromising perimeter.

DataShield runs as a local OS-level proxy. Every request is intercepted, fingerprinted, and either neutralized or poisoned — in microseconds, with zero cloud round-trips.

ENGINE · A

Data Poisoner

Injects high-entropy pseudo-random gibberish into outbound telemetry streams. Trackers receive realistic-looking noise indistinguishable from genuine traffic — corrupting their models without breaking site functionality.

04 / FAQ

Questions, answered.

Does DataShield phone home?

No. The binary contains zero network code outside the local proxy. The verification harness in our repo proves this — every release is tested to confirm outbound bytes equal zero over a 30-second sniff window.

Will it break the websites I visit?

Rarely. Data Poisoner only targets known telemetry endpoints; first-party functionality is untouched. If a site misbehaves, one-click bypass from the menu bar suspends protection for that domain for 10 minutes.

Is the MIT license really the whole thing?

Yes. No "Enterprise Edition", no paid tier, no upsell. The same code that powers Fortune-500 deployments is the code you install. We're funded by individual sponsorships and foundation grants — not your data.

How is this different from uBlock Origin?

uBlock runs inside the browser and blocks at the DOM level. DataShield runs at the OS network layer — it covers every app on your machine (browsers, Electron, mobile simulators, CLI tools) and additionally poisons telemetry that slips past blockers.
